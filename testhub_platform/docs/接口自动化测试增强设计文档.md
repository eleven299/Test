# 接口自动化测试增强设计文档

> 版本: v1.0 (草案)
> 编写日期: 2026-06-18
> 适用模块: `apps/api_testing/`

## 1. 背景与目标

### 1.1 背景

`apps/api_testing/` 已具备完整的接口管理与自动化骨架(接口、集合、环境、套件、定时任务、Allure 报告、通知),但执行引擎存在多处关键缺陷,使其难以支撑真实业务场景的接口自动化。

最严重的问题:**前一个接口的响应无法影响后一个接口**,这意味着"登录拿 token → 后续接口用 token"这类最基本的链路场景无法实现。

### 1.2 目标

将 `api_testing` 模块从"接口集合执行器"升级为"生产级接口自动化引擎",对齐 Postman/HttpRunner/Apifox 的核心能力,具体包括:

- **变量提取与上下文传递**:支持从响应中提取变量供后续请求使用
- **前后置脚本**:激活已有的 `pre_request_script` / `post_request_script` 字段
- **断言系统增强**:扩展操作符、统一套件级与请求级断言
- **执行引擎重构**:支持重试、超时、fail-fast、思考时间
- **数据驱动测试(DDT)**:同一接口用数据集跑 N 遍
- **WebSocket 执行**:落地 WS 类型接口的执行逻辑
- **CLI/CI 入口**:支持命令行触发,接入 Jenkins/GitLab CI

### 1.3 非目标

- 不重写前端 UI 框架,只做必要的交互扩展
- 不替换现有的定时任务调度框架(APScheduler)
- 不引入新的消息队列(Celery 已在依赖中,但不强制启用)
- 不破坏现有 API 兼容性

### 2.2 关键缺陷(详细)

#### 🔴 缺陷 1: 无变量提取/上下文传递(致命)

**证据**: `views.py:682-871` (套件 execute 方法)

```python
# 每个请求都重新构造 variables,只从 environment 取
for suite_request in suite_requests:
    variables = {}
    if test_suite.environment:
        variables.update(test_suite.environment.variables)
    # ... 执行请求 ...
    # ❌ 没有从响应中提取任何变量回写到 variables
```

**后果**:
- 无法实现"登录 → 提取 token → 后续接口鉴权"
- 无法实现"创建订单 → 提取订单号 → 查询订单"
- 无法实现"列表接口 → 取首条 ID → 查询详情"

#### 🔴 缺陷 2: 前后置脚本字段未实现

**证据**:
- `models.ApiRequest` 定义了 `pre_request_script` / `post_request_script` 两个 TextField
- `grep -n "pre_request_script\|post_request_script" views.py` 返回空

**后果**:字段是摆设,前端如果让用户填了脚本也根本不会执行。

#### 🟡 缺陷 3: 套件级断言未走统一断言引擎

**证据**: `views.py:782-790`

```python
# 套件请求的断言只处理 status_code 一种
for assertion in suite_request.assertions:
    if assertion.get('type') == 'status_code':
        # ...只比对状态码
```

**后果**:用户在套件级别配置的 `json_path` / `contains` / `header` 断言全部失效。

#### 🟡 缺陷 4: 超时硬编码 30s

**证据**: `views.py:440, 763` — `timeout=30` 写死。

**后果**:慢接口直接超时失败。

#### 🟡 缺陷 5: 无失败重试机制

**证据**: 套件执行循环中无 retry 计数器。

**后果**:网络抖动、偶发 5xx 直接判失败。

#### 🟡 缺陷 6: 断言操作符单一

**证据**: `utils.py` 中
- `status_code` / `header` / `equals`: 只有 `==`
- `contains`: 只有 `in`
- `json_path`: 只有 `str(actual) == str(expected)`

**后果**:无法表达 `!=` `>` `<` `>=` `<=` `regex` `not_contains` `in_array` `length_eq` 等。

#### 🟡 缺陷 7: 无 fail-fast / 思考时间

**证据**: 套件执行 `for suite_request in suite_requests:` 一路跑到底。

**后果**:某关键前置接口(如登录)失败时,后续依赖它的接口仍会执行,浪费时间且报告噪声大。

#### 🟡 缺陷 8: WebSocket 接口无法执行

**证据**: `models.ApiRequest.REQUEST_TYPE_CHOICES` 含 `WEBSOCKET`,但两个 execute 方法中没有任何 `if request_type == 'WEBSOCKET'` 分支。

**后果**:WS 类型接口创建后无法运行。

#### 🟡 缺陷 9: 无数据驱动

**证据**: `TestSuiteRequest` 只有 `assertions` 字段,无数据集关联。

**后果**:无法实现"登录接口用 5 组账号各跑一遍"。

#### 🟡 缺陷 10: 无 CLI/CI 入口

**证据**: 触发执行的唯一方式是 HTTP API (`POST /api/test-suites/{id}/execute/`)。

**后果**:无法在 CI 流水线中通过命令行触发,只能依赖 Web 触发或定时任务。

## 3. 设计原则

1. **向后兼容**:所有新增字段提供默认值,不破坏现有数据;现有 API 行为不变。
2. **单一执行引擎**:单接口、套件、定时任务、CLI 共用同一套执行核心,避免逻辑分叉。
3. **变量作用域清晰**:区分环境变量、提取变量、迭代变量、全局变量,优先级明确。
4. **可观测**:执行过程产出结构化日志与逐步结果,便于排查。
5. **可扩展**:断言、提取器、脚本运行时通过注册机制扩展,不硬编码。

---

## 4. 详细设计

### 4.1 变量作用域模型

定义四级变量作用域,优先级从低到高:

| 层级 | 来源 | 生命周期 | 覆盖关系 |
|---|---|---|---|
| L1 全局变量 | `Environment` scope=GLOBAL | 整个执行周期 | 最低 |
| L2 环境变量 | `Environment` scope=LOCAL | 整个执行周期 | 覆盖 L1 |
| L3 提取变量 | 前一请求的 `extractors` 输出 | 当前套件执行内,跨请求传递 | 覆盖 L2 |
| L4 迭代变量 | DDT 数据集当前行 | 单次迭代内 | 覆盖 L3 |
| L5 临时变量 | 前置脚本 `vars.xxx = ...` | 当前请求内 | 最高 |

引用语法统一为 `{{var_name}}`(环境/提取)与 `${func(args)}`(动态函数),两者可嵌套,如 `Bearer {{token}}` `${md5({{timestamp}})}`。

### 4.3 变量提取器设计

新建 `apps/api_testing/extractors.py`:

```python
class VariableExtractor:
    """从响应中提取变量"""

    def extract(self, response, extractors, context):
        """
        response: requests.Response 对象
        extractors: 提取规则列表
        context: 当前变量上下文(用于写入)
        """
        for rule in extractors:
            try:
                value = self._extract_one(response, rule)
                scope = rule.get('target_scope', 'suite')
                context.set(rule['name'], value, scope=scope)
            except Exception as e:
                # 提取失败不阻塞流程,记录 warning
                context.log('extract_failed', rule=rule['name'], error=str(e))
                if 'default' in rule:
                    context.set(rule['name'], rule['default'], scope=scope)

    def _extract_one(self, response, rule):
        source = rule['source']
        expr = rule['expression']

        if source == 'json_body':
            from jsonpath_ng import parse
            matches = parse(expr).find(response.json())
            if not matches:
                raise ValueError(f"JSONPath 无匹配: {expr}")
            return matches[0].value

        elif source == 'header':
            return response.headers.get(expr)

        elif source == 'status_code':
            return response.status_code

        elif source == 'regex':
            import re
            m = re.search(expr, response.text)
            if not m:
                raise ValueError(f"正则无匹配: {expr}")
            group = rule.get('group', 0)
            return m.group(group)

        elif source == 'raw_body':
            return response.text

        elif source == 'xml_body':
            from lxml import etree
            tree = etree.fromstring(response.content)
            results = tree.xpath(expr)
            return results[0] if results else None

        raise ValueError(f"未知 source: {source}")
```

### 4.4 前后置脚本运行时

#### 4.4.1 Python 运行时(默认,推荐)

提供受限沙箱环境,注入常用对象:

```python
# apps/api_testing/script_runtime.py
ALLOWED_BUILTINS = {
    'len': len, 'str': str, 'int': int, 'float': float,
    'bool': bool, 'list': list, 'dict': dict, 'range': range,
    'min': min, 'max': max, 'sum': sum, 'abs': abs,
    'round': round, 'sorted': sorted, 'enumerate': enumerate,
    'zip': zip, 'map': map, 'filter': filter,
}

class ScriptContext:
    """暴露给脚本的上下文对象"""
    def __init__(self, variables, request_data, response=None):
        self.vars = variables          # 可读可写: vars.token = "xxx"
        self.request = request_data    # 请求对象(只读)
        self.response = response       # 响应对象(后置脚本可用,只读)
        self.log = []                  # 脚本日志

class PythonScriptRuntime:
    def execute(self, script, context):
        sandbox_globals = {
            '__builtins__': ALLOWED_BUILTINS,
            'ctx': context,
            # 常用工具
            'json': __import__('json'),
            're': __import__('re'),
            'time': __import__('time'),
            'datetime': __import__('datetime').datetime,
            'hashlib': __import__('hashlib'),
            'base64': __import__('base64'),
        }
        # 限制 exec 权限:无 import 语句(通过 ast 预检)
        self._validate_script(script)
        exec(script, sandbox_globals)
```

脚本示例(后置 - 提取 token 并做签名):

```python
# 自动注入 ctx
import json
data = ctx.response.json()
ctx.vars.token = data['data']['access_token']

# 为下一个请求预计算签名
import hashlib, time
ts = str(int(time.time()))
sign = hashlib.md5(f"{ctx.vars.token}{ts}{'secret'}".encode()).hexdigest()
ctx.vars.timestamp = ts
ctx.vars.sign = sign
```

**安全**:通过 AST 预检禁止 `import` / `open` / `eval` / `exec` / `__import__` / `os` / `subprocess` 等危险调用。

```python
class TestExecutionEngine:
    """接口自动化执行引擎"""

    def __init__(self, execution, user=None):
        self.execution = execution          # TestExecution 记录
        self.user = user
        self.context = VariableContext()    # 变量上下文(L1-L5)
        self.stop_requested = False

    def run(self):
        """主入口"""
        self._init_context()
        suite = self.execution.test_suite

        try:
            # 加载环境变量到 L1/L2
            self._load_environment(suite.environment)

            # 执行前可注入全局前置脚本
            self._run_suite_setup()

            # DDT: 计算迭代次数
            iterations = self._compute_iterations(suite)

            for iter_idx in range(iterations):
                if self.stop_requested:
                    break

                self.context.push_iteration(iter_idx)
                self._load_iteration_data(iter_idx)

                for step in suite.steps.enabled_ordered():
                    if self.stop_requested:
                        break

                    result = self._run_step(step)

                    if not result.passed and step.is_critical and suite.fail_fast:
                        self.execution.stop_reason = f"关键步骤 {step.request.name} 失败"
                        self.stop_requested = True
                        break

                    # 思考时间
                    if suite.think_time > 0:
                        time.sleep(suite.think_time / 1000)

                self.context.pop_iteration()

            self._run_suite_teardown()
            self._finalize_execution()

        except Exception as e:
            self.execution.status = 'FAILED'
            self.execution.stop_reason = str(e)
            self._finalize_execution()

    def _run_step(self, step):
        """执行单个步骤(含重试、超时、断言、提取)"""
        request = step.request
        last_result = None

        max_attempts = 1 + request.retry_count
        for attempt in range(1, max_attempts + 1):
            try:
                # 1. 解析变量
                resolved = self._resolve_request(request)

                # 2. 执行前置脚本
                self._run_script(request.pre_request_script,
                                 ScriptContext(self.context, resolved))

                # 3. 再次解析(脚本可能改了变量)
                resolved = self._resolve_request(request)

                # 4. 发起请求(含超时)
                response = self._send_request(resolved,
                                              timeout=step.timeout_override or request.timeout)

                # 5. 构造结果对象
                result = TestStepResult(...)
                result.attempt = attempt

                # 6. 执行断言(请求级 + 套件级合并)
                assertions = list(request.assertions) + list(step.assertions)
                result.assertions_results = run_assertions(response, assertions, self.context)
                result.passed = all(a['passed'] for a in result.assertions_results)

                # 7. 失败时重试
                if not result.passed and attempt < max_attempts:
                    time.sleep(request.retry_interval)
                    continue

                # 8. 通过或最后一次:执行提取
                if result.passed or attempt == max_attempts:
                    VariableExtractor().extract(
                        response, request.extractors, self.context
                    )

                # 9. 执行后置脚本
                self._run_script(request.post_request_script,
                                 ScriptContext(self.context, resolved, response))

                # 10. 保存步骤结果
                result.save()
                return result

            except Exception as e:
                last_result = self._make_error_result(step, e, attempt)
                if attempt < max_attempts:
                    time.sleep(request.retry_interval)
                    continue
                last_result.save()
                return last_result

        return last_result
```

#### 4.6.1 HTTP 与 WebSocket 分发

```python
def _send_request(self, resolved, timeout):
    if resolved.request_type == 'WEBSOCKET':
        return self._send_websocket(resolved, timeout)
    return self._send_http(resolved, timeout)

def _send_http(self, resolved, timeout):
    import requests
    return requests.request(
        method=resolved.method,
        url=resolved.url,
        headers=resolved.headers,
        params=resolved.params,
        json=resolved.body if resolved.body_type == 'json' else None,
        data=resolved.body if resolved.body_type in ('raw', 'form-data') else None,
        timeout=timeout,
        verify=not resolved.skip_ssl_verify,
    )

def _send_websocket(self, resolved, timeout):
    """WebSocket 请求执行"""
    import websocket
    import json as json_mod

    ws = websocket.create_connection(
        resolved.url,
        header=[f"{k}: {v}" for k, v in resolved.headers.items()],
        timeout=timeout
    )
    try:
        # 发送消息(body 即要发送的消息)
        if resolved.body:
            ws.send(json_mod.dumps(resolved.body) if isinstance(resolved.body, (dict, list))
                    else str(resolved.body))

        # 接收响应(可配置接收条数与等待时间)
        messages = []
        for _ in range(resolved.ws_receive_count or 1):
            try:
                msg = ws.recv()
                messages.append(msg)
            except websocket.WebSocketTimeoutException:
                break

        return WebSocketResponse(status_code=101, messages=messages, headers={})
    finally:
        ws.close()
```

`WebSocketResponse` 是一个 duck-type 兼容 `requests.Response` 的对象(有 `.text` `.json()` `.headers`),让断言与提取器代码无需特判。

```python
def _normalize_assertion(assertion):
    """旧格式转新格式"""
    if 'operator' in assertion:
        return assertion  # 新格式

    legacy_type = assertion.get('type')
    mapping = {
        'status_code':   {'source': 'status_code', 'operator': 'equals'},
        'contains':      {'source': 'raw_body',   'operator': 'contains'},
        'json_path':     {'source': 'json_body',  'operator': 'equals',
                          'expression': assertion.get('json_path')},
        'header':        {'source': 'header',     'operator': 'equals',
                          'expression': assertion.get('header_name')},
        'response_time': {'source': 'response_time', 'operator': 'less_equal'},
        'equals':        {'source': 'raw_body',   'operator': 'equals'},
    }
    return {**assertion, **mapping.get(legacy_type, {})}
```

## 8. 风险评估与对策

| 风险 | 等级 | 对策 |
|---|---|---|
| 脚本执行的安全风险(代码注入) | 🔴 高 | AST 预检 + 受限 builtins + 独立进程可选(后续用 subprocess 隔离) |
| 重构破坏现有定时任务调度 | 🟡 中 | 引擎接口对齐现有 `_execute_test_suite` 签名,定时任务调用点平滑切换 |
| 大响应体撑爆数据库 | 🟡 中 | `response_snapshot` 超过阈值(如 1MB)截断 + 异步落盘到文件 |
| 迁移失败影响线上 | 🟡 中 | 先在 staging 验证;migration 拆分(字段与索引分开提交) |
| 变量提取死循环(变量互相引用) | 🟢 低 | 解析时记录依赖栈,检测循环 |
| WebSocket 长连接阻塞调度线程 | 🟡 中 | WS 执行必须显式 timeout,默认 30s |
